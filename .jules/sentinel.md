## 2024-05-24 - Secure File Creation Pattern (TOCTOU Prevention)
**Vulnerability:** A Time-of-Check to Time-of-Use (TOCTOU) vulnerability existed in `src/logger.py` where a log file was created using `Path.touch(exist_ok=True)` and subsequently its permissions were changed using `os.chmod(self.log_file, 0o600)`.
**Learning:** This approach leaves a brief window between the file's creation and the permissions change where a malicious actor could access or modify the file.
**Prevention:** Instead of separating the creation and permission modification, use `os.open` with the `os.O_CREAT` flag and directly specify the desired permissions (e.g., `0o600`). This ensures the file is created atomically with the correct permissions. For example: `fd = os.open(file_path, os.O_CREAT | os.O_APPEND | os.O_WRONLY, 0o600)` followed by `os.close(fd)`.

## 2024-05-24 - NaN/Inf Bounds Checking Bypass Vulnerability
**Vulnerability:** Input validation logic in `src/validation.py` for numeric values like quantity, price, stop price, and percentage only verified if the input was a valid float and checked against hard limits (e.g., `qty < OrderValidator.MIN_QUANTITY`). Passing `float('nan')` or `float('inf')` circumvented these checks because comparisons with `nan` generally evaluate to `False`.
**Learning:** Standard comparison operators (`<`, `>`, `<=`, `>=`) can be unreliable or behave unexpectedly when dealing with special floating-point values like `NaN` and `Infinity`. This could allow users to bypass business logic boundaries or trigger application crashes/unexpected behavior in downstream components.
**Prevention:** Always validate that numeric floating-point input is finite before applying boundary logic. Use `math.isfinite(value)` from the Python standard library to explicitly reject `nan`, `inf`, and `-inf` inputs before further processing or comparisons.
