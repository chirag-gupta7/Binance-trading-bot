## 2024-03-10 - Secure file permissions for log files
**Vulnerability:** TOCTOU (Time-of-Check to Time-of-Use) Vulnerability in log file creation. The logger in `src/logger.py` used `Path(self.log_file).touch()` to create the file and subsequently called `os.chmod(self.log_file, 0o600)`. Because these are two separate system calls, there is a race condition window where the file is created with wide default permissions (e.g. 0o644 based on umask) before it is locked down. This could temporarily expose sensitive trading data.
**Learning:** Python's `open()` or `Path.touch()` use the system's default `umask` to dictate default file creation permissions, which often result in read access for other users. Calling `chmod()` after file creation introduces a race condition that can be exploited by an attacker monitoring file creations.
**Prevention:** To prevent this, always create sensitive files securely by using `os.open` with the desired flags (such as `os.O_CREAT | os.O_WRONLY`) and passing the explicit secure mode (e.g., `0o600`) as the third argument. This guarantees the file is atomicly created with secure permissions.
