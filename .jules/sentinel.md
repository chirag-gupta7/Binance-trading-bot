## 2024-05-24 - API Client Hanging Risk and Local Data Leaks
**Vulnerability:** External API calls to Binance Futures did not have an explicit timeout configured in the `UMFutures` initialization, risking Denial of Service (DoS) due to hanging connections. Additionally, local log files for the CLI trading bot did not enforce strict file permissions, leaving sensitive trading data vulnerable to unauthorized local access.
**Learning:** Default configurations in external libraries (like `UMFutures`) may lack essential security measures like timeouts, leading to potential hanging issues. When logging sensitive operations (e.g., trading activities, API keys), it's crucial to explicitly manage file permissions to prevent data leakage in shared or multi-user environments.
**Prevention:** Always specify an explicit `timeout` when initializing external API clients. For file handling involving sensitive data, explicitly set strict permissions (`0o600` for files, `0o700` for directories) before writing or appending to prevent local unauthorized access.
