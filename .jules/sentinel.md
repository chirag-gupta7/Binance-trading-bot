## 2024-03-04 - [Trading Data Exposure & API Denial of Service]
**Vulnerability:** The trading bot logged sensitive financial and trading data (e.g. trade orders, potentially API keys if debug turned on) to a globally readable `bot.log` file. Also, the external API calls to Binance Futures (`UMFutures` client) lacked an explicit timeout setting.
**Learning:** This architectural security gap means log files created by the application implicitly inherit broad permissions (typically `0o644`) exposing trading data to any user on the system. Concurrently, omitted timeouts open up the application to hanging processes and DoS when the external provider is slow or drops connections, which can be critical for time-sensitive trading operations.
**Prevention:** Always explicitly set restrictive permissions (e.g. `0o600`) when creating log files containing sensitive data. Always configure an explicit timeout parameter (e.g. `timeout=10`) when initializing external HTTP clients or API SDKs.
